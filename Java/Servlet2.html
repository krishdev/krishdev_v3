Can servlet have a constructor and why
Yes, it will be used by the container for instantiation
Servlet Config object
Servlet config is specific to each servlet. We can pass parameters to the servlet using the servlet config object <servlet>
<servlet-name>ServletName</servlet-name>
<servlet-class>com.mkyong.ServletDemo</servlet-class>
<init-param>
<param-name>email</param-name>
<param-value>admin@email.com</param-value>
</init-param>
</servlet>
And in the servlet's init method or any doget or dopost method, call getServletConfig().getInitParam(email");
Servlet Context object
It is common to all the servlets
<context-param>
<param-name>globalVariable</param-name>
<param-value>javapapers.com</param-value>
</context-param>
getServletConfig().getServletContext().getInitParam()
Difference between forward and sendRedirect
Forward - servlet forwards the request to a different servlet (browser has no idea of this and there is no url change)
SendRedirect - Servlet asks the browser to redirect the request to a different url, in this case the url changes and browser is aware of the redirect
Difference between forward and include
Include - get the response content from other servlet and include it in the current servlet response
forward - just forwards the request to a new servlet, current servlet response is lost
What is lazy-loading/pre-initialization of servlet?
lazy-loading - servlet is not loaded in the beginning, it is loaded only when it receives the first request
pre-initialization - servlet is loaded when the application begins. Can be achieved by using element
What is session and why should we track it
Different requests from the same browser/user has to be treated as single session. This is essential in order to maintain the state of the user. Ex. shopping cart, login/logout status etc.,
Means of Session tracking
URL Rewriting, cookies, hidden form objects, SSL sessions
Cookies
User data which are not very sensitive can be saved as cookies in the users browser. Session data can be stored as cookie which needs no server interaction
URL Re-writing
extra data is attached to the URL everytime, session is identified based on the extra data in the url
How to destroy session?
session.invalidate()
