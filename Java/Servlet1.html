What are Servlets and their advantages
Servlets are server side java components. They receive the request from HTTP, process it and render the response back to the client
Advantages of Servers over CGI processes
Servlets run within the same address space of the server. So it does not require extra memory
In Servlets, each request is a thread. Where as in CGI its a process. Thread performance is much better than process.
All features of java available like security, platform independent and oops
Servlet Life cycle
Load - Servlet is loaded when the application begins (can specify using load-on-startup) or when the servlet is called for the first time.
Init - the init() method is called after the servlet is loaded, servlet enters the service only after it successfully executes the init method. It is called only once during the life time of servlet
service - based on the request, either doGet(), doPut(), doDelete() or doPost() method is invoked. For each request, the web container spawns a new request. Override doGet or doPost, cannot override service method
destroy - when the server is no longer in service or when server shutsdown, the destroy method is called. It is called only once for a servlet, after it is called, the servlet is marked for garbage collection
Web Container
Web Container is a part of web server. It handles the request and response from the servlet
1. It loads the servlet
2. It handles the communication between servlet and the server
3. It spawns a new thread for each request
4. It delegates requests to the right servlet
How to configure a servlet in web.xml file
<servlet>
<servlet-name>HelloWorld</servlet-name>
<servlet-class>HelloWorld</servlet-class>
</servlet>
<servlet-mapping>
<servlet-name>HelloWorld</servlet-name>
<url-pattern>/HelloWorld</url-pattern>
</servlet-mapping>
What is the difference between get and post method
Get - all data is passed thru' url rewriting, so we can pass only less data (1024 char) and data that is not very sensitive. We can bookmark the request. generally used to get some data from server
Post - data is packed and sent to the servlet as a seperate message. no restriction on amount of data to pass, can pass sensitive data, cannot bookmark the request. Generally used when there is something to update
